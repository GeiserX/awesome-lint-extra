# CLAUDE.md — awesome-lint-extra

## Overview
Custom linter for awesome lists that validates entry format, alphabetical order, badge presence, URL hosts, and table of contents consistency. Designed for lists with advanced formatting (clickable badges, custom tags) that standard awesome-lint does not support.

## Tech Stack
- Python (linting logic)
- PyPI package (`awesome-lint-extra`)
- GitHub Actions (usable as an action via `action.yml`)
- pytest (testing)
- Codecov (coverage)

## Development
```bash
# Install from PyPI
pip install awesome-lint-extra

# Run the linter
awesome-lint-extra

# Run tests
pytest

# Use as GitHub Action
# uses: GeiserX/awesome-lint-extra@main
```

## Architecture
- `lint.py` — Core linting engine
- `action.yml` — GitHub Action definition
- `.awesomerc.example.json` — Example configuration file
- `tests/` — Test suite
- `media/` — Images and assets
- `docs/` — Documentation

## Key Rules
- Validates: entry format, description style, alphabetical order, no duplicate URLs, URL host allowlist, badge presence, ToC consistency
- Configuration via `.awesomerc.json` in the target repo
- Exit code 0 = all checks pass, non-zero = lint errors found
- Published to PyPI as `awesome-lint-extra`
